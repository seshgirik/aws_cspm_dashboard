ÔøΩ Key Differences Between Access Paths
üîµ PATH A - Laptop (Public Internet):
‚úì IAM Policy check ONLY
‚úó Security Groups do NOT apply
‚úó VPC Endpoint Policy NOT evaluated

Result: Laptop users can access Bedrock if IAM allows, regardless of VPC restrictions.
üü† PATH B - EC2 (VPC Endpoint):
‚úì Security Group check (ENI traffic)
‚úì VPC Endpoint Policy (role/IP)
‚úì IAM Policy (model access)

Result: EC2 must pass ALL three security layers to access Bedrock.
ÔøΩ Why This Matters:
The AllowedBedrockGroup role restrictions only work for EC2 instances inside the VPC because they rely on VPC Endpoint policies and Security Groups.

When a laptop user accesses Bedrock via the public AWS API endpoint, the traffic doesn't go through the VPC Endpoint, so those network-level controls are completely bypassed.

This is a key distinction in AWS security architecture!
