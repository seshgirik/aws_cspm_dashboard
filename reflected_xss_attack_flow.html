üéØ Common Reflected XSS Vectors

Search Functionality

Attacker injects payload in search query:

/search?q=<img src=x onerror=alert(1)>

Error Pages

Payload in error message parameters:

/error?msg=<script>...</script>

Comments/Feedback

XSS in user comments displayed immediately:

/comment?text=<iframe src=evil.com>

URL Fragments

Payload in URL hash (sometimes reflected):

/page#search=<svg onload=alert(1)>

HTTP Headers

Reflected in pages showing HTTP info:

User-Agent: <script>alert(1)</script>

Redirect Parameters

Payload in redirect URL displayed:

/redirect?url=javascript:alert(1)

üìä Reflected vs Stored XSS

Reflected XSS (This Demo)

  • ‚úì Payload in URL/request
  • ‚úì Not stored on server
  • ‚úì User must click malicious link
  • ‚úì Only affects that user
  • ‚úì Easier to notice (URL visible)
  • ‚úì Each attack requires new URL

Stored XSS (More Dangerous)

  • ‚úì Payload stored in database
  • ‚úì Executes when others view content
  • ‚úì No link click needed
  • ‚úì Affects ALL users viewing it
  • ‚úì Harder to detect
  • ‚úì Single attack, multiple victims

üõ°Ô∏è Prevention Strategies

Output Encoding

// PHP - htmlspecialchars()
echo htmlspecialchars($user_input, ENT_QUOTES, 'UTF-8');

// JavaScript - textContent instead of innerHTML
element.textContent = userInput;

// HTML entity encoding
< becomes &lt;
> becomes &gt;
" becomes &quot;

Input Validation

// Whitelist approach
allowed_chars = /^[a-zA-Z0-9\s]+$/
if (!input.match(allowed_chars)) {
  reject_input()
}

// Content Security Policy (CSP)
Content-Security-Policy: 
  script-src 'self'

Other Prevention Methods

1. Content Security Policy

Restricts where scripts can be loaded from, preventing inline script execution.

2. HTTP-Only Cookies

Prevents JavaScript from accessing session cookies, making theft harder.

3. Same-Site Cookies

Restricts cookie sending in cross-site requests, limiting CSRF exploitation.

4. Security Headers

X-Frame-Options, X-Content-Type-Options prevent related attacks.

5. Web Application Firewall

Detects and blocks common XSS patterns at network level.

6. Security Testing

Regular penetration testing and vulnerability scanning with tools like OWASP ZAP.

üìã Quick Reference

What Makes Reflected XSS Possible?

  1. User input is taken from URL/request
  2. No sanitization/escaping of the input
  3. Input is reflected directly in HTML response
  4. Browser executes any script tags in the HTML
  5. Attacker's code runs in victim's browser context
